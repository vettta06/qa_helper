# QA Helper — Система анализа требований, тест-кейсов и баг-репортов

> Реализовано в рамках кейса:  
 **«Анализ требований, оформление тест-кейсов и баг-репортов»**

Данный проект представляет собой **веб-систему для QA-инженера**, позволяющую:
- Формализовать **функциональные и нефункциональные требования** к продукту (в данном случае — холодильнику),
- Создавать **тест-кейсы**, привязанные к требованиям,
- Оформлять **баг-репорты** при выявлении несоответствий.

---

## Возможности системы

- **Требования**: функциональные и нефункциональные, с указанием типа устройства
- **Тест-кейсы**: привязаны к требованиям, содержат пошаговую инструкцию
- **Баг-репорты**:  
  - Уровень серьёзности: `low`, `medium`, `high`, `critical`  
  - Полное описание: шаги, окружение, ожидаемое/фактическое поведение  
  - Связь с тест-кейсом
- **Поддержка устройств**: холодильник, стиральная машина, духовка, микроволновка
- **Веб-интерфейс**:  
  - Создание, просмотр, удаление сущностей  
  - Страницы деталей (`/bug/1`, `/requirement/2` и т.д.)
- **Экспорт в CSV**: отдельно для требований, тестов и багов

##  Запуск

### Требования
- Python 3.10+
- pip

### Инструкция

```bash
git clone https://github.com/vettta06/qa_helper.git
pip install -r requirements.txt
uvicorn app.main:app --reload
```
### Доступ
- Веб-интерфейс: http://localhost:8000
- API Docs: http://localhost:8000/docs
